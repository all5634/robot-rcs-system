# RCMS 开发落地清单（不改需求版）

## 1. 当前已完成（M1）
- 建立三层结构：Controller、Service、Client。
- 完成 7 个核心接口路径与参数校验。
- 实现统一响应结构：code、message、data。
- 实现全局异常处理（参数错误/业务错误/系统错误）。
- 实现内存态机器人管理：注册、心跳、状态更新、列表/详情查询。
- 实现定时离线检测（按心跳超时阈值）。
- 实现控制指令基础状态流转（QUEUED -> DISPATCHED -> SUCCESS/FAILED）。
- 预留 DataClient 外部数据服务接口，当前使用 MockDataClient 日志占位。

## 2. 下一阶段（M2）
- 接入真实外部数据服务（替换 MockDataClient）：
  - saveRobot
  - saveHeartbeat
  - saveStatus
  - saveCommand
  - updateCommandStatus
- 增加接口级审计日志：请求时间、robotId/commandId、结果码、耗时。
- 补齐统一错误码字典（400x/404x/500x）。
- 增加基于 DTO 的字段边界与语义校验。

## 3. 稳定性与高可用（M3）
- 增加外部服务故障处理：重试、超时、熔断、降级。
- 明确多实例一致性策略：
  - 方案 A：心跳/在线状态由外部服务作为唯一真值源。
  - 方案 B：引入共享缓存并定义写入策略。
- 命令链路补齐幂等：commandId 幂等键、重复投递去重。
- 引入线程池治理：队列容量、拒绝策略、监控指标。

## 4. 测试与验收（M4）
- 单元测试：Service 核心逻辑覆盖。
- 集成测试：7 个接口 happy path + invalid path。
- 性能压测：心跳/状态上报 + 查询并发场景。
- 稳定性测试：长稳运行（>=72h）和外部依赖抖动场景。

## 5. 部署交付（M5）
- 增加容器化文件（Dockerfile）与部署参数说明。
- 增加环境配置模板（dev/test/prod）。
- 增加启动、健康检查、日志采集文档。
